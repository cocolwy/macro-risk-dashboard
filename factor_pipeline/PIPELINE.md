# 因子挖掘流水线（公共协议）

每个新因子必须按 **S0 → S7** 顺序推进。可并行探索，但**不得跳过预注册直接下交易结论**。

状态约定：`pending` → `in_progress` → `done` → `blocked` / `skipped`

---

## S0 · 目标锚定（Goal）

**产出：** 一句话交易目标 + 标的 + 持仓 horizon

| 注意点 | 说明 |
|--------|------|
| 先定目标再定检验 | 「研究 VIX 是否上涨」≠「用 UVIX 赚钱」；标的错位会导致整条链路结论不可用 |
| 写清可交易工具 | 指数（^VIX）通常不可直接交易；ETF/ETN（UVIX）有杠杆、衰减、跟踪误差 |
| 写清动作 | 做多 / 做空 / 择时加仓 / 仅风控过滤 |
| 写清持有期 | 隔夜、T-N~T-1、事件日日内等；horizon 决定窗口与成本假设 |

**门禁：** 未写清「标的 + 动作 + horizon」不得进入 S1。

---

## S1 · 经济直觉（Hypothesis）

**产出：** 可证伪的一句话假设 + 机制故事

| 注意点 | 说明 |
|--------|------|
| 机制优先于统计 | 「谁在什么约束下被迫交易，留下什么痕迹？」写不出则先别挖 |
| 区分水平 vs 变化 | VIX 水平高 ≠ VIX 会继续涨；交易 UVIX 通常关心**变化方向** |
| 写清失效条件 | 例如：政策高度可预期、效应已被套利、仅高/低波动 regime 成立 |

**门禁：** 假设必须可被数据否定（不能事后改口成「本来就是条件成立」）。

---

## S2 · 预注册协议（Pre-register）

**产出：** 写进 case JSON 的 `preregister` 字段，**再跑主检验**

| 注意点 | 说明 |
|--------|------|
| 固定主窗口 | 主结论只用预注册窗口；扫窗口只能进探索章节 |
| 固定样本与基准 | 样本起止、非事件基准、稀疏采样（防自相关） |
| 固定显著性规则 | 单侧/双侧、多重比较（Bonferroni 等）、α |
| 探索与确认隔离 | 探索结果不得直接升格为「可交易结论」 |

**门禁：** `preregister` 未填写完整，主检验结果一律标为探索性。

---

## S3 · 数据与实现（Data）

**产出：** 数据字典 + 可复现脚本路径 + 无 lookahead 声明

| 注意点 | 说明 |
|--------|------|
| 时间戳对齐 | 宏观发布时刻 vs 日线收盘；用错时点会虚增效应 |
| 指数 ≠ 产品 | 用 ^VIX 做研究时，必须单列「映射到 UVIX 的缺口」 |
| 修订与初值 | CPI 等有修订；若策略用实时发布，勿用最终修订序列回测 |
| 复现路径 | 脚本、JSON 输出、看板页面三者可追溯 |

---

## S4 · 探索性分析（Explore）

**产出：** 候选窗口 / 条件拆分 / 可视化；全部标记 `exploratory: true`

| 注意点 | 说明 |
|--------|------|
| 多重检验 | 扫 4 个窗口 × 3 事件 ≈ 假阳性放大器；报告时必须警告 |
| 不挑最优当主结论 | 「最优窗口」只能作为下一轮预注册候选 |
| 样本外预留 | 探索用过的样本，确认阶段尽量用独立区间 |

---

## S5 · 确认检验（Confirm）

**产出：** 预注册窗口上的主结论 + 敏感性（子样本 / COVID / 发表前后）

| 注意点 | 说明 |
|--------|------|
| 只报告预注册检验为「主结论」 | 探索显著 ≠ 确认显著 |
| 效应衰减检查 | 论文发表后、近 1–2 年是否消失（套利 / regime 变化） |
| 条件分析要诚实 | 低 VIX 显著若来自同一探索样本，需独立期外再确认 |

---

## S6 · 可交易性（Tradability）— 目标若是 UVIX，此步不可跳过

**产出：** 从「指数统计显著」到「产品能否赚钱」的桥接结论

| 注意点 | 说明 |
|--------|------|
| 产品结构 | UVIX 跟踪 VIX 期货（常有 contango），长期持有有衰减；短持也有杠杆噪声 |
| 信号→仓位映射 | VIX 上涨概率高，不自动等于 UVIX 多头期望收益为正 |
| 成本 | 买卖价差、隔夜、杠杆再平衡；事件策略样本少，成本敏感 |
| 容量与执行 | 日线信号 vs 需要的入场时刻（T-5 开盘？T-3 收盘？） |
| 风险 | 事件落空时 UVIX 可能剧烈回撤；需最大回撤 / 尾部情景 |

**门禁：** 未完成 S6，状态不得标 `confirmed`；最多 `conditional` / `weak`。

---

## S7 · 归档与看板（Archive）

**产出：** Alpha Deck 条目 + case JSON +（可选）实验看板链接

| 注意点 | 说明 |
|--------|------|
| 否定也要归档 | Dead 因子是资产，避免三个月后重复发明 |
| 状态诚实 | confirmed / conditional / weak / dead / pending |
| next_steps 可执行 | 每条下一步必须能对应新的 S2 预注册 |

---

## 状态到流水线阶段映射（建议）

| Alpha Deck status | 通常停在 |
|-------------------|----------|
| `pending` | S0–S2 |
| `weak` | S5 主检验弱 / S6 不可交易 |
| `conditional` | S5 仅部分 regime 成立，S6 未完成或半完成 |
| `confirmed` | S5 确认 + S6 可交易桥接通过 |
| `dead` | S5/S6 明确否定，归档 |
